Databases & SQL

In IGCSE 0478/0984 you write single-table SQL queries using SELECT, FROM, WHERE, ORDER BY, the aggregate functions and (in the 2026 syllabus) GROUP BY — applied to a flat table given in the question.

Database Terminology

Vocab

Examiners deduct marks for sloppy use of "row" when you mean "record", or "field" when you mean "data type". Learn these precisely.

Table

A structured collection of data, organised in rows and columns. Each table stores information about one type of entity (e.g. PRODUCT, STUDENT, BOOK).

Record (row)

A single entry in a table — one entity, e.g. one product. All the values for one item, read left to right.

Field (column)

A single attribute — one property of an entity, e.g. Price. Every record has a value for every field (possibly NULL).

Primary key

A field (or combination of fields) that uniquely identifies each record. Two rules: no duplicates and never NULL. Example: ProductID.

Foreign key

A field in one table that references the primary key of another table, linking the two. IGCSE 2026 is mostly single-table, but the vocabulary is examinable.

Database

A collection of related tables, organised so that data can be efficiently stored, retrieved and updated. Managed by software called a DBMS.

Tip: in exam answers always say "record" not "row" and "field" not "column" — those are the marker's keywords.

Data Types in Databases

Types

When you design a table you must give each field an appropriate data type. The IGCSE syllabus uses the same names you'll find in real SQL.

Data typeStoresExample value
TEXT / VARCHAR(n) Strings — variable-length characters up to n "Laptop X1"
INTEGER Whole numbers (positive or negative, no decimal point) 12
REAL / DECIMAL / FLOAT Decimal (real) numbers — money, measurements, averages 899.99
DATE Calendar dates in YYYY-MM-DD form "2026-05-19"
BOOLEAN Logical values — only TRUE or FALSE TRUE
TIME Times of day in HH:MM:SS form (less common in IGCSE) "14:30:00"

Choosing the right type matters: storing Price as TEXT means you can't add it up; storing Stock as REAL allows nonsensical "12.7 units".

SQL Queries — Build-Up

Each card shows a query in increasing complexity. Click "What this returns" to reveal the expected result rows.

Basic SELECT

1 — Select all
Return every field of every record. * is the wildcard meaning "all fields".
SELECT * FROM PRODUCT;
What this returns

All 10 records with all 7 fields — the entire PRODUCT table unchanged.

2 — Specific fields
List the column names you want, separated by commas.
SELECT Name, Price FROM PRODUCT;
What this returns

10 rows, each showing only Name and Price: Laptop X1 / 899.99, Coffee Beans 1kg / 12.50, Bluetooth Headphones / 79.00, … Standing Desk / 449.00.

3 — DISTINCT
Eliminate duplicate values — useful for getting the list of unique categories, suppliers, etc.
SELECT DISTINCT Category FROM PRODUCT;
What this returns

3 rows: Electronics, Grocery, Furniture — the unique category values, no repeats.

WHERE clause — filtering records

4 — Simple WHERE
Filter by an exact text match. Text values must be in quotes.
SELECT * FROM PRODUCT
WHERE Category = "Electronics";
What this returns

4 records: P001 Laptop X1, P003 Bluetooth Headphones, P006 Smartphone Z9, P009 USB Cable.

5 — Numeric comparison
Compare numeric fields with <, >, <=, >=. Numbers are not quoted.
SELECT Name, Price FROM PRODUCT
WHERE Price < 50;
What this returns

5 rows where Price < 50: Coffee Beans 1kg / 12.50, Tea Bags 100ct / 4.99, Desk Lamp / 39.99, Olive Oil 500ml / 8.99, USB Cable / 5.99.

6 — Inequality
<> is the standard SQL "not equal to" operator. Use it to exclude a value.
SELECT Name FROM PRODUCT
WHERE Stock <> 0;
What this returns

9 names — every product except Bluetooth Headphones (the only one whose Stock = 0).

Combining conditions — AND, OR

7 — AND
AND means both conditions must be true.
SELECT * FROM PRODUCT
WHERE Category = "Grocery"
  AND Price < 10;
What this returns

2 records: P005 Tea Bags 100ct (4.99) and P008 Olive Oil 500ml (8.99). Coffee Beans is Grocery but costs 12.50 so it's excluded.

8 — OR
OR means at least one condition is true.
SELECT * FROM PRODUCT
WHERE Category = "Electronics"
   OR Category = "Furniture";
What this returns

7 records — every product except the 3 Grocery items (Coffee Beans, Tea Bags, Olive Oil).

9 — AND + OR with parentheses
Mix AND/OR carefully — use parentheses to force the evaluation order. Without them, AND binds tighter than OR and your query may filter the wrong rows.
SELECT * FROM PRODUCT
WHERE Available = TRUE
  AND (Price < 50 OR Stock > 100);
What this returns

Available items that are either cheap (< 50) or abundantly stocked (> 100): P002 Coffee Beans (price 12.50), P005 Tea Bags (price 4.99 and stock 120), P007 Desk Lamp (39.99), P008 Olive Oil (8.99), P009 USB Cable (5.99, stock 200) — 5 records.

Pattern matching — LIKE

10 — LIKE with %
% matches any number of characters (including zero).
SELECT Name FROM PRODUCT
WHERE Name LIKE "S%";
-- names starting with capital S
What this returns

2 rows: Smartphone Z9, Standing Desk.

11 — LIKE with _
_ matches exactly one character. The pattern "P00_" is 4 characters: P, 0, 0, then any single character.
SELECT * FROM PRODUCT
WHERE ProductID LIKE "P00_";
What this returns

9 records — P001 through P009. P010 is excluded because its ID is 4 characters but the last is not a single-position match (the IDs are length 4, so this actually returns all 10? Note: ProductID values are P001…P010, all length 4, so P00_ matches P001–P009 and P0__ would be needed for P010.) The result is 9 rows (P001–P009).

12 — LIKE with both sides %
%text% matches anything containing the substring. LIKE is normally case-insensitive in IGCSE answers.
SELECT * FROM PRODUCT
WHERE Name LIKE "%oil%";
What this returns

1 record: Olive Oil 500ml — the only name containing the letters "oil".

Ranges and lists — BETWEEN, IN

13 — BETWEEN
BETWEEN x AND y is shorthand for >= x AND <= y — inclusive at both ends.
SELECT Name, Price FROM PRODUCT
WHERE Price BETWEEN 5 AND 50;
What this returns

4 rows: Coffee Beans 1kg / 12.50, Desk Lamp / 39.99, Olive Oil 500ml / 8.99, USB Cable / 5.99. (Tea Bags at 4.99 is just below 5, so excluded.)

14 — IN
IN (list) is shorthand for chained ORs — match any value in the list.
SELECT * FROM PRODUCT
WHERE Supplier IN ("TechCorp", "BeanCo");
What this returns

4 records: P001 Laptop X1, P002 Coffee Beans, P005 Tea Bags, P006 Smartphone Z9.

Sorting — ORDER BY

15 — Ascending
ASC = smallest → largest (or A → Z). It's the default.
SELECT Name, Price FROM PRODUCT
ORDER BY Price ASC;
What this returns

10 rows in price order: Tea Bags / 4.99, USB Cable / 5.99, Olive Oil / 8.99, Coffee Beans / 12.50, Desk Lamp / 39.99, Bluetooth Headphones / 79.00, Office Chair / 249.99, Standing Desk / 449.00, Smartphone Z9 / 599.00, Laptop X1 / 899.99.

16 — Descending
DESC = largest → smallest. Useful for "top N" style queries.
SELECT Name, Stock FROM PRODUCT
ORDER BY Stock DESC;
What this returns

10 rows in stock order, highest first: USB Cable / 200, Tea Bags / 120, Olive Oil / 75, Coffee Beans / 50, Desk Lamp / 30, Smartphone Z9 / 25, Laptop X1 / 12, Office Chair / 8, Standing Desk / 3, Bluetooth Headphones / 0.

17 — Multiple sort keys
List sort fields in priority order. The second key only matters when the first key ties.
SELECT * FROM PRODUCT
ORDER BY Category ASC, Price DESC;
What this returns

All 10 rows grouped by category alphabetically (Electronics, Furniture, Grocery), and inside each group sorted by Price highest-first. So Electronics: Laptop X1 (899.99), Smartphone Z9 (599.00), Bluetooth Headphones (79.00), USB Cable (5.99); Furniture: Standing Desk (449.00), Office Chair (249.99), Desk Lamp (39.99); Grocery: Coffee Beans (12.50), Olive Oil (8.99), Tea Bags (4.99).

Aggregate functions

Aggregates take many rows and return one number. They go inside SELECT, not in WHERE.

18 — COUNT(*)
Counts the number of records in the result. * counts rows regardless of NULLs.
SELECT COUNT(*) FROM PRODUCT;
What this returns

1 value: 10 — the total number of products in the table.

19 — COUNT with WHERE
Filter first with WHERE, then count the remaining rows.
SELECT COUNT(*) FROM PRODUCT
WHERE Category = "Electronics";
What this returns

1 value: 4 (Laptop X1, Bluetooth Headphones, Smartphone Z9, USB Cable).

20 — SUM
Adds up the values of a numeric field.
SELECT SUM(Stock) FROM PRODUCT;
What this returns

1 value: 523 = 12 + 50 + 0 + 8 + 120 + 25 + 30 + 75 + 200 + 3.

21 — SUM with expression
You can apply arithmetic inside an aggregate — here we calculate the total value of stock on hand (price × quantity) for available products only.
SELECT SUM(Price * Stock) FROM PRODUCT
WHERE Available = TRUE;
What this returns

1 value — the total £-value of available stock: 899.99×12 + 12.50×50 + 249.99×8 + 4.99×120 + 599.00×25 + 39.99×30 + 8.99×75 + 5.99×200 (skipping P003 and P010 which are FALSE).

22 — AVG
Mean (average) of a numeric field. Equivalent to SUM / COUNT.
SELECT AVG(Price) FROM PRODUCT;
What this returns

1 value: about 234.94 — sum of all 10 prices divided by 10.

23 — MIN / MAX
Smallest and largest values. You can select multiple aggregates in one query.
SELECT MIN(Price), MAX(Price) FROM PRODUCT;
What this returns

1 row, 2 values: 4.99 (Tea Bags) and 899.99 (Laptop X1).

GROUP BY (new in 2026 syllabus)

GROUP BY splits the table into groups based on a field, then applies an aggregate per group. You get one row per distinct group value.

Rule: every field in the SELECT list must either be the field you're grouping by, or be inside an aggregate function.

24 — GROUP BY with COUNT
Count records in each category.
SELECT Category, COUNT(*)
FROM PRODUCT
GROUP BY Category;

DDL — Data Definition Language

Schema commands

SQL is split into two main parts:

  • DDL — Data Definition Language: defines and changes the structure of tables. Commands: CREATE TABLE, ALTER TABLE, DROP TABLE.
  • DML — Data Manipulation Language: works with the data in tables. Commands: SELECT, INSERT, UPDATE, DELETE (this is what we've been doing).
CREATE TABLE

Defines a new table: name, fields, data types, and constraints such as PRIMARY KEY and NOT NULL.

ALTER TABLE

Modifies an existing table — typically ADD a new field, change a data type, or drop a column.

DROP TABLE

Permanently deletes the table and all its data. Irreversible — examiners love asking why this is dangerous.

Example — creating the PRODUCT table

CREATE TABLE PRODUCT (
  ProductID TEXT PRIMARY KEY,
  Name TEXT NOT NULL,
  Category TEXT,
  Price REAL,
  Stock INTEGER,
  Available BOOLEAN
);

Notes: each field is name TYPE [constraints], separated by commas; PRIMARY KEY follows the type of the field that is the key; NOT NULL forces a value to be supplied; a semicolon ends the statement.

SQL Cheat Sheet

One-glance reference

Anatomy of a SELECT query

SELECT <fields>
FROM <table>
WHERE <condition>
GROUP BY <field>
ORDER BY <field> ASC|DESC;

Clauses always appear in this order. Only SELECT and FROM are required; the rest are optional.

Aggregate functions
  • SUM(field) — total of numeric values
  • COUNT(*) — number of records
  • AVG(field) — mean
  • MIN(field) — smallest value
  • MAX(field) — largest value
Wildcards (with LIKE)
  • % — any sequence of characters (zero or more)
  • _ — exactly one character
  • "%abc%" — contains "abc"
  • "abc%" — starts with "abc"
  • "%abc" — ends with "abc"
Comparison & logical operators
  • = equals (single =)
  • <> not equal to
  • < > <= >= ordering
  • AND OR NOT — boolean connectives
  • IN (...), BETWEEN x AND y, LIKE pattern
Common keywords (uppercase by convention)
  • SELECT FROM WHERE — retrieve
  • ORDER BY ASC DESC — sort
  • GROUP BY — categorise
  • DISTINCT — remove duplicates
  • CREATE / ALTER / DROP TABLE — schema

Common SQL Mistakes

Lose-mark traps
MistakeWrongCorrect
Forgetting quotes around text values — every year examiners report this as the most common error. WHERE Category = Electronics WHERE Category = "Electronics"
Using == for equality (a habit from Python/Java). SQL uses a single =. WHERE Price == 50 WHERE Price = 50
Using != instead of <>. <> is the standard, exam-safe operator; != works in some dialects only. WHERE Stock != 0 WHERE Stock <> 0
Clauses in the wrong order — ORDER BY must come after WHERE. SELECT * FROM PRODUCT ORDER BY Price WHERE Stock > 0; SELECT * FROM PRODUCT WHERE Stock > 0 ORDER BY Price;
Missing semicolon at the end of a query — some markschemes accept this, but the safe habit is to always include it. SELECT * FROM PRODUCT SELECT * FROM PRODUCT;
Putting an aggregate inside WHERE — illegal. Aggregates go in SELECT (or filtered with HAVING after GROUP BY). WHERE SUM(Stock) > 100 SELECT SUM(Stock) FROM ... GROUP BY ...;
Quoting numbers — numbers are never quoted; quotes turn them into strings, breaking comparisons. WHERE Price < "50" WHERE Price < 50
Using LIKE without wildcards — works, but is exactly equivalent to =, so pointless. WHERE Name LIKE "Laptop X1" WHERE Name LIKE "Laptop%" (or use =)

Read your query back out loud word-by-word before submitting — most syntax slips show up immediately.

Multi-part Exam Question (~12 marks)

Practice — full question

The table PRODUCT shown earlier is stored in a database. Write SQL queries to answer each of the following. Each part is worth 2 marks.

  1. (a) Output the names of all products priced above £100.
  2. (b) Output the total stock value (Price × Stock summed) for all available products.
  3. (c) Output the supplier with the highest total stock. (Use GROUP BY.)
  4. (d) Output all products in the "Electronics" category, sorted by price from highest to lowest.
  5. (e) Output the average price of products in each category.
  6. (f) Output all products whose name contains the word "Bean" or "Coffee".
Model answers (click to reveal)

(a) Names of products priced above £100 — 2 marks

SELECT Name
FROM PRODUCT
WHERE Price > 100;

Marks: 1 for SELECT Name FROM PRODUCT; 1 for the correct WHERE Price > 100. Result: Laptop X1, Office Chair, Smartphone Z9, Standing Desk (4 rows).

(b) Total stock value of available products — 2 marks

SELECT SUM(Price * Stock)
FROM PRODUCT
WHERE Available = TRUE;

Marks: 1 for the SUM(Price * Stock) aggregate; 1 for the WHERE Available = TRUE filter. Excludes P003 and P010 which are FALSE.

(c) Supplier with the highest total stock — 2 marks

SELECT Supplier, SUM(Stock)
FROM PRODUCT
GROUP BY Supplier
ORDER BY SUM(Stock) DESC;

Marks: 1 for the GROUP BY Supplier; 1 for the ORDER BY SUM(Stock) DESC. The first row of the output is the answer — AudioPlus, with 200 units (Bluetooth Headphones 0 + USB Cable 200). Showing the first row only would normally be done with LIMIT 1, but IGCSE does not require LIMIT — the ordered list is accepted.

(d) Electronics products sorted by price descending — 2 marks

SELECT *
FROM PRODUCT
WHERE Category = "Electronics"
ORDER BY Price DESC;

Marks: 1 for the WHERE Category = "Electronics"; 1 for ORDER BY Price DESC. Result order: Laptop X1 (899.99), Smartphone Z9 (599.00), Bluetooth Headphones (79.00), USB Cable (5.99).

(e) Average price per category — 2 marks

SELECT Category, AVG(Price)
FROM PRODUCT
GROUP BY Category;

Marks: 1 for the AVG(Price) aggregate selected alongside Category; 1 for the GROUP BY Category. Result: Electronics ≈ 395.99, Furniture ≈ 246.33, Grocery ≈ 8.83.

(f) Products whose name contains "Bean" or "Coffee" — 2 marks

SELECT *
FROM PRODUCT
WHERE Name LIKE "%Bean%"
   OR Name LIKE "%Coffee%";

Marks: 1 for using LIKE with the %...% wildcards on both sides; 1 for combining the two patterns with OR. Result: P002 Coffee Beans 1kg (matches both patterns — returned once).

Exam strategy: write the query on the dotted lines exactly as you would in code — keywords in CAPITALS, text in double quotes, numbers without quotes, a semicolon at the end. Don't write English sentences explaining the query unless the question asks for an explanation.

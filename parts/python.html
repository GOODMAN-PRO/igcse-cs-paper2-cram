Python ↔ Pseudocode Equivalents

Paper 2 will ask you to read and write code in either Cambridge pseudocode or a high-level language (Python is the most common). Almost every pseudocode construct has a near-direct Python twin — the trick is remembering the small differences (indexing, casting, indentation, colons). Learn this table cold and you can translate any pseudocode answer into Python (or back) under exam pressure.

Translation Syntax Paper 2

Full translation table

Pseudocode Python Note
INPUT X X = input() input() always returns a string — cast if you want a number.
INPUT N (integer) N = int(input()) Wrap in int(...). Will crash on non-numeric input.
INPUT R (real) R = float(input()) Use float for decimals.
OUTPUT X print(X) print auto-adds a newline.
OUTPUT X, Y, Z print(X, Y, Z) Python inserts a space between items; pseudocode concatenates.
X ← 5 X = 5 Single = means assign. (The arrow is U+2190.)
X ← Y + 1 X = Y + 1 Right-hand side evaluated first.
IF cond THEN ... ELSE ... ENDIF if cond: / elif: / else: Colon required; indentation defines the block.
CASE OF X ... ENDCASE if/elif/else chain (or match) Cambridge accepts if/elif as the Python equivalent of CASE.
FOR i ← 1 TO 10 for i in range(1, 11): Python's upper bound is exclusive — add 1.
FOR i ← 10 TO 1 STEP -1 for i in range(10, 0, -1): Counting down: stop is one past the last value.

Three traps that cost marks

  1. Forgetting to cast input(). Writing age = input() then if age > 18: raises TypeError: '>' not supported between instances of 'str' and 'int'. Fix: age = int(input()).
  2. Off-by-one in range(). range(1, 10) produces 1, 2, ..., 9 — it does NOT include 10. For pseudocode FOR i ← 1 TO 10 write range(1, 11).
  3. 0-indexed lists vs 1-indexed pseudocode arrays. Names[1] in pseudocode is the first item; in Python it is the second. Either translate by subtracting 1, or pad your list with a dummy at index 0.

More traps (bonus)

  1. Confusing = (assign) with == (compare). Pseudocode uses = for both!
  2. Forgetting the : at the end of if / for / while / def lines.
  3. Indentation matters in Python — it defines the block. Mixing tabs and spaces causes IndentationError.
  4. print(a, b) puts a space between values; pseudocode OUTPUT a, b concatenates with no separator.

Arrays and Lists

In IGCSE pseudocode an array is a fixed-size, 1-indexed collection of items of the same type; a Python list is 0-indexed, can change size, and may hold mixed types.

Data structures 1D & 2D

1D arrays side-by-side

Pseudocode
DECLARE Names : ARRAY[1:5] OF STRING
Names[1] ← "Sam"
Names[2] ← "Alex"
FOR i ← 1 TO 5
  OUTPUT Names[i]
NEXT i
Python
names = [""] * 5             # 5 empty slots
names[0] = "Sam"          # index shifts -1
names[1] = "Alex"
for i in range(5):
    print(names[i])

2D arrays side-by-side

Pseudocode
DECLARE Grid : ARRAY[1:3, 1:3] OF INTEGER
FOR r ← 1 TO 3
  FOR c ← 1 TO 3
    Grid[r, c] ← 0
  NEXT c
NEXT r
Python
grid = [[0]*3 for _ in range(3)]
for r in range(3):
    for c in range(3):
        grid[r][c] = 0
# access with grid[row][col]

Common array operations

Traverse (print every item)
FOR i ← 1 TO 5
  OUTPUT Names[i]
NEXT i
for n in names:
    print(n)
Linear search
Found ← FALSE
FOR i ← 1 TO 5
  IF Names[i] = Target THEN
    Found ← TRUE
  ENDIF
NEXT i
OUTPUT Found
found = False
for n in names:
    if n == target:
        found = True
print(found)
Modify (overwrite an element)
Scores[3] ← 100
scores[2] = 100   # index shifts -1
Sum all values
Total ← 0
FOR i ← 1 TO 20
  Total ← Total + Scores[i]
NEXT i
OUTPUT Total
total = 0
for s in scores:
    total += s
print(total)
# or: print(sum(scores))
Find maximum value
Max ← Scores[1]
FOR i ← 2 TO 20
  IF Scores[i] > Max THEN
    Max ← Scores[i]
  ENDIF
NEXT i
OUTPUT Max
mx = scores[0]
for s in scores[1:]:
    if s > mx:
        mx = s
print(mx)
# or: print(max(scores))

Exam-style questions

1. An array Scores[1:20] stores 20 game scores. Write pseudocode to output the highest score and how many times it appears. 6 marks

Mark-scheme moves: initialise max + count, loop once to find the max, loop again (or use the same pass) to count occurrences, output both.

Max ← Scores[1]
FOR i ← 2 TO 20
  IF Scores[i] > Max THEN
    Max ← Scores[i]
  ENDIF
NEXT i

Count ← 0
FOR i ← 1 TO 20
  IF Scores[i] = Max THEN
    Count ← Count + 1
  ENDIF
NEXT i

OUTPUT "Highest: ", Max
OUTPUT "Appears: ", Count, " times"

Marks: initialise Max [1] · correct loop bounds 1–20 [1] · compare and update Max [1] · initialise + increment Count [1] · correct comparison [1] · sensible output of both values [1].

2. Write a Python function that takes a list of integers and returns the average. 4 marks 
def average(nums):
    if len(nums) == 0:           # avoid div-by-zero
        return 0
    total = 0
    for n in nums:
        total += n
    return total / len(nums)

Marks: correct def with parameter [1] · running total [1] · divide by length [1] · return the value (not print) [1].

Procedures and Functions

Procedures and functions are named, reusable blocks of code. They help with decomposition — breaking a big problem into smaller, testable pieces.

Decomposition Parameters

Procedure vs Function

Aspect Procedure Function
Purpose Performs an action (prints, modifies data, …). Calculates and returns a value.
Returns a value? No. Yes — via RETURN.
Pseudocode keyword PROCEDURE ... ENDPROCEDURE FUNCTION ... RETURNS T ... ENDFUNCTION
How to call CALL ProcName(args) Use in an expression: x ← FnName(args)
Python def name(params): — no return (or bare return). def name(params): with return value.

Procedure example

Pseudocode
PROCEDURE PrintRow(Width : INTEGER)
  FOR i ← 1 TO Width
    OUTPUT "*"
  NEXT i
ENDPROCEDURE

CALL PrintRow(5)
Python
def print_row(width):
    for i in range(width):
        print("*", end="")
    print()        # move to next line

print_row(5)

Function example

Pseudocode
FUNCTION Square(N : INTEGER) RETURNS INTEGER
  RETURN N * N
ENDFUNCTION

Answer ← Square(7)
Python
def square(n):
    return n * n

answer = square(7)
print(answer)   # 49

Parameters, arguments, and scope

  • Parameter — the variable in the function/procedure definition. Example: Width in PROCEDURE PrintRow(Width : INTEGER), or n in def square(n):.
  • Argument — the actual value passed in when the procedure or function is called. Example: the 5 in CALL PrintRow(5), or the 7 in square(7).
  • Local variable — declared inside the function/procedure. Only exists while that block is running; it cannot be seen from outside.
  • Global variable — declared at the top level of the program; visible everywhere. In Python you can read a global inside a function, but to modify it you must write global name inside the function first.

Why use procedures and functions?

  • Code reuse — write once, call many times.
  • Easier to test individual pieces — supports decomposition.
  • More readable — a meaningful name describes a block of logic at a glance.
  • Less duplication — one place to fix a bug instead of many.
  • Easier maintenance — changes are contained; less risk of breaking unrelated code.
  • Team work — different programmers can write different procedures in parallel.

Multiple parameters & multiple return values

Pseudocode
FUNCTION MinMax(A : ARRAY[1:10] OF INTEGER)
        RETURNS INTEGER
  Lo ← A[1]
  Hi ← A[1]
  FOR i ← 2 TO 10
    IF A[i] < Lo THEN Lo ← A[i] ENDIF
    IF A[i] > Hi THEN Hi ← A[i] ENDIF
  NEXT i
  RETURN Hi - Lo          // range
ENDFUNCTION
Python (returns a tuple of both)
def min_max(a):
    lo = a[0]
    hi = a[0]
    for n in a[1:]:
        if n < lo: lo = n
        if n > hi: hi = n
    return lo, hi      # tuple

low, high = min_max([3, 7, 1, 9, 4])
print(low, high, high - low)

Exam-style questions

1. Write a pseudocode function CheckRange that takes an integer N and two integers Low and High as parameters, and returns TRUE if N is between Low and High inclusive, FALSE otherwise. 5 marks 
FUNCTION CheckRange(N : INTEGER,
                    Low : INTEGER,
                    High : INTEGER) RETURNS BOOLEAN
  IF N >= Low AND N <= High THEN
    RETURN TRUE
  ELSE
    RETURN FALSE
  ENDIF
ENDFUNCTION

Marks: FUNCTION header with three parameters [1] · RETURNS BOOLEAN [1] · correct condition using AND [1] · inclusive comparison (>= and <=) [1] · returns TRUE/FALSE appropriately [1].

2. State three reasons why programmers use procedures and functions in their code. 3 marks
  • Reusability — the same block can be called many times without rewriting.
  • Decomposition / readability — breaks a big problem into small named pieces, easier to follow.
  • Easier testing / maintenance — each procedure can be tested on its own; a fix in one place propagates.

Any three sensible reasons gain a mark each. Other valid answers: reduces duplication, supports teamwork, easier debugging.

File Handling

Files give programs persistent storage — data survives after the program ends. For Paper 2 you need to read, write, and append to plain text files in both pseudocode and Python. Always remember to open → use → close.

I/O Text files

Reading a file

Pseudocode
OPENFILE "scores.txt" FOR READ
WHILE NOT EOF("scores.txt") DO
  READFILE "scores.txt", Line
  OUTPUT Line
ENDWHILE
CLOSEFILE "scores.txt"
Python
f = open("scores.txt", "r")
for line in f:
    print(line.strip())   # strip removes \n
f.close()

Idiomatic Python uses a with block (with open("scores.txt") as f: ...) which auto-closes the file even on error. IGCSE mark schemes accept either, but typically expect explicit open and close.

Writing to a file

Pseudocode
OPENFILE "log.txt" FOR WRITE
WRITEFILE "log.txt", "Run complete"
CLOSEFILE "log.txt"
Python
f = open("log.txt", "w")
f.write("Run complete\n")   # add \n yourself!
f.close()

Opening with "w" overwrites any existing file with the same name. Be careful — data already in the file is lost.

Appending

Pseudocode
OPENFILE "log.txt" FOR APPEND
WRITEFILE "log.txt", "Another entry"
CLOSEFILE "log.txt"
Python
f = open("log.txt", "a")   # a = append
f.write("Another entry\n")
f.close()

"a" mode adds to the end of the file and creates it if missing — existing data is kept.

Worked example: sum numbers from a file

File data.txt has one integer per line. Read them all and output the total.

Pseudocode
OPENFILE "data.txt" FOR READ
Total ← 0
WHILE NOT EOF("data.txt") DO
  READFILE "data.txt", Num
  Total ← Total + Num
ENDWHILE
CLOSEFILE "data.txt"
OUTPUT Total
Python
total = 0
f = open("data.txt", "r")
for line in f:
    total += int(line.strip())   # cast string to int
f.close()
print(total)

Common file errors

  • Forgetting to close the file. Data written may stay in a buffer and never reach disk. Always f.close() (or use a with block).
  • Reading a missing file. open("missing.txt", "r") raises FileNotFoundError. Check existence first or wrap in try / except.
  • Forgetting \n when writing. Python's write() does not auto-append a newline — without it everything ends up on one line.
  • Reading numbers without casting. line is a string. "5" + 3 raises TypeError. Use int(line.strip()) or float(line.strip()).
  • Opening with "w" when you meant "a". "w" wipes existing content — a frequent and painful mistake.

Exam-style questions

1. A text file grades.txt contains one mark per line. Write a Python program that reads all marks and outputs the average. 6 marks 
f = open("grades.txt", "r")
total = 0
count = 0
for line in f:
    total += int(line.strip())   # cast each mark
    count += 1
f.close()

if count > 0:
    average = total / count
    print("Average:", average)
else:
    print("No data")

Marks: open file for reading [1] · loop through every line [1] · cast string to int [1] · running total + counter [1] · close the file [1] · output total divided by count [1].

2. Write pseudocode that reads names from a file friends.txt and writes them in reverse order to reversed.txt. 5 marks 
DECLARE Names : ARRAY[1:100] OF STRING
Count ← 0

// 1. Read all names into the array
OPENFILE "friends.txt" FOR READ
WHILE NOT EOF("friends.txt") DO
  Count ← Count + 1
  READFILE "friends.txt", Names[Count]
ENDWHILE
CLOSEFILE "friends.txt"

// 2. Write them out in reverse
OPENFILE "reversed.txt" FOR WRITE
FOR i ← Count TO 1 STEP -1
  WRITEFILE "reversed.txt", Names[i]
NEXT i
CLOSEFILE "reversed.txt"

Marks: open input for read + counter logic [1] · read every line into the array [1] · open output file for write [1] · loop from Count down to 1 (STEP -1) [1] · write each name and close both files [1].

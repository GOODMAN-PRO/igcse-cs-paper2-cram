Flowcharts

Flowcharts visualise algorithms during the Design stage — every box, arrow and diamond maps to a step in your pseudocode.

Symbols Reference Card

6 shapes — memorise them

Each shape has one job. Use the right shape or lose easy marks.

START / STOP
Terminator
Oval / rounded rectangle. Every flowchart begins with START and ends with STOP.
Total ← 0
Process
Rectangle. Used for assignment or calculation, e.g. Total ← Total + N.
INPUT / OUTPUT
Input / Output
Parallelogram. Used for INPUT X and OUTPUT Y.
X > 5?
Decision
Diamond. Contains a condition — splits flow into exactly two paths (Yes/No or True/False).
A
Connector
Small circle, labelled A, B, C… Joins flowchart sections (or continues onto another page).
→
Arrow / Flow line
Shows direction. Usually top → bottom and left → right. Loops go back upwards.
Exam tip: if you draw the wrong shape (e.g. a rectangle around INPUT N) you usually lose the symbol mark even if the logic is right.

10 Worked examples

Study the diagram first, then expand the pseudocode to check the mapping shape-by-shape.

1. Average of 10 numbers

Count-controlled loop

Input ten numbers, accumulate the total, then output the mean.

flowchart TD A([START]) --> B[Total ← 0] B --> C[Count ← 1] C --> D{Count <= 10?} D -- Yes --> E[/INPUT Number/] E --> F[Total ← Total + Number] F --> G[Count ← Count + 1] G --> D D -- No --> H[Average ← Total / 10] H --> I[/OUTPUT Average/] I --> J([STOP])
Show equivalent pseudocode 
DECLARE Total, Number, Count, Average : INTEGER
Total ← 0
FOR Count ← 1 TO 10
  INPUT Number
  Total ← Total + Number
NEXT Count
Average ← Total / 10
OUTPUT Average

2. Validated password

Post-condition loop

Keep asking until the user types the correct password "secret".

flowchart TD A([START]) --> B[/INPUT Password/] B --> C{Password = 'secret'?} C -- No --> D[/OUTPUT 'Incorrect, try again'/] D --> B C -- Yes --> E[/OUTPUT 'Access granted'/] E --> F([STOP])
Show equivalent pseudocode 
DECLARE Password : STRING
REPEAT
  INPUT Password
  IF Password <> "secret" THEN
    OUTPUT "Incorrect, try again"
  ENDIF
UNTIL Password = "secret"
OUTPUT "Access granted"

3. Linear search

Flag + loop

Search an array List[1..10] for Target; stop early if found.

flowchart TD A([START]) --> B[/INPUT Target/] B --> C[Found ← False] C --> D[Index ← 1] D --> E{Index <= 10 AND Found = False?} E -- Yes --> F{List[Index] = Target?} F -- Yes --> G[Found ← True] G --> H[Position ← Index] H --> I[Index ← Index + 1] F -- No --> I I --> E E -- No --> J{Found = True?} J -- Yes --> K[/OUTPUT Position/] J -- No --> L[/OUTPUT 'Not found'/] K --> M([STOP]) L --> M
Show equivalent pseudocode 
DECLARE List : ARRAY[1:10] OF INTEGER
DECLARE Target, Index, Position : INTEGER
DECLARE Found : BOOLEAN
INPUT Target
Found ← FALSE
Index ← 1
WHILE Index <= 10 AND Found = FALSE
  IF List[Index] = Target THEN
    Found ← TRUE
    Position ← Index
  ENDIF
  Index ← Index + 1
ENDWHILE
IF Found = TRUE THEN OUTPUT Position
ELSE OUTPUT "Not found"
ENDIF

4. Largest of three numbers

Nested IF

Read three values; output the largest using two comparisons.

flowchart TD A([START]) --> B[/INPUT A, B, C/] B --> D{A >= B?} D -- Yes --> E{A >= C?} D -- No --> F{B >= C?} E -- Yes --> G[Largest ← A] E -- No --> H[Largest ← C] F -- Yes --> I[Largest ← B] F -- No --> H G --> J[/OUTPUT Largest/] H --> J I --> J J --> K([STOP])
Show equivalent pseudocode 
DECLARE A, B, C, Largest : INTEGER
INPUT A, B, C
IF A >= B THEN
  IF A >= C THEN Largest ← A
  ELSE Largest ← C
  ENDIF
ELSE
  IF B >= C THEN Largest ← B
  ELSE Largest ← C
  ENDIF
ENDIF
OUTPUT Largest

5. Count evens in 20 inputs

MOD operator

Read 20 numbers; count how many are even using N MOD 2 = 0.

flowchart TD A([START]) --> B[Count ← 0] B --> C[I ← 1] C --> D{I <= 20?} D -- Yes --> E[/INPUT N/] E --> F{N MOD 2 = 0?} F -- Yes --> G[Count ← Count + 1] G --> H[I ← I + 1] F -- No --> H H --> D D -- No --> J[/OUTPUT Count/] J --> K([STOP])
Show equivalent pseudocode 
DECLARE N, Count, I : INTEGER
Count ← 0
FOR I ← 1 TO 20
  INPUT N
  IF N MOD 2 = 0 THEN
    Count ← Count + 1
  ENDIF
NEXT I
OUTPUT Count

6. Factorial of N

Accumulating product

Input N, compute N! = 1×2×3×…×N. Start at 1 (multiplicative identity).

flowchart TD A([START]) --> B[/INPUT N/] B --> C[Fact ← 1] C --> D[I ← 1] D --> E{I <= N?} E -- Yes --> F[Fact ← Fact * I] F --> G[I ← I + 1] G --> E E -- No --> H[/OUTPUT Fact/] H --> J([STOP])
Show equivalent pseudocode 
DECLARE N, Fact, I : INTEGER
INPUT N
Fact ← 1
FOR I ← 1 TO N
  Fact ← Fact * I
NEXT I
OUTPUT Fact

7. Times table for N

FOR loop

Input N; output the lines 1×N through 12×N.

flowchart TD A([START]) --> B[/INPUT N/] B --> C[I ← 1] C --> D{I <= 12?} D -- Yes --> E[Result ← I * N] E --> F[/OUTPUT I, ' x ', N, ' = ', Result/] F --> G[I ← I + 1] G --> D D -- No --> H([STOP])
Show equivalent pseudocode 
DECLARE N, I, Result : INTEGER
INPUT N
FOR I ← 1 TO 12
  Result ← I * N
  OUTPUT I, " x ", N, " = ", Result
NEXT I

8. Celsius → Fahrenheit (validated)

Range check

Input must be ≥ −273°C (absolute zero). Re-prompt until valid.

flowchart TD A([START]) --> B[/INPUT C/] B --> D{C >= -273?} D -- No --> E[/OUTPUT 'Invalid temperature'/] E --> B D -- Yes --> F[F ← C * 9 / 5 + 32] F --> G[/OUTPUT F/] G --> H([STOP])
Show equivalent pseudocode 
DECLARE C, F : REAL
REPEAT
  INPUT C
  IF C < -273 THEN
    OUTPUT "Invalid temperature"
  ENDIF
UNTIL C >= -273
F ← C * 9 / 5 + 32
OUTPUT F

9. ATM withdrawal

Compound validation

Amount must be > 0 and ≤ balance. If valid, deduct and show new balance.

flowchart TD A([START]) --> B[/INPUT Balance/] B --> C[/INPUT Amount/] C --> D{Amount > 0?} D -- No --> E[/OUTPUT 'Amount must be positive'/] E --> Z([STOP]) D -- Yes --> F{Amount <= Balance?} F -- No --> G[/OUTPUT 'Insufficient funds'/] G --> Z F -- Yes --> H[Balance ← Balance - Amount] H --> I[/OUTPUT 'New balance: ', Balance/] I --> Z
Show equivalent pseudocode 
DECLARE Balance, Amount : REAL
INPUT Balance, Amount
IF Amount <= 0 THEN
  OUTPUT "Amount must be positive"
ELSE
  IF Amount > Balance THEN
    OUTPUT "Insufficient funds"
  ELSE
    Balance ← Balance - Amount
    OUTPUT "New balance: ", Balance
  ENDIF
ENDIF

10. Bubble sort — one outer pass

Swap logic

A single sweep over List[1..5] swapping adjacent items if out of order.

flowchart TD A([START]) --> B[I ← 1] B --> C{I <= 4?} C -- Yes --> D{List[I] > List[I+1]?} D -- Yes --> E[Temp ← List[I]] E --> F[List[I] ← List[I+1]] F --> G[List[I+1] ← Temp] G --> H[I ← I + 1] D -- No --> H H --> C C -- No --> J[/OUTPUT List/] J --> K([STOP])
Show equivalent pseudocode 
DECLARE List : ARRAY[1:5] OF INTEGER
DECLARE I, Temp : INTEGER
FOR I ← 1 TO 4
  IF List[I] > List[I+1] THEN
    Temp ← List[I]
    List[I] ← List[I+1]
    List[I+1] ← Temp
  ENDIF
NEXT I
OUTPUT List

How to draw a flowchart

Exam method
  • · Always begin with START and end with STOP — both in ovals.
  • · Use arrows to show direction. Default flow is top → bottom, left → right; loops curve back upwards.
  • · Every decision diamond splits into exactly two paths, labelled Yes/No or True/False.
  • · Inputs and outputs always use the parallelogram. Don't put them in a rectangle.
  • · Calculations and assignments use the rectangle: Total ← Total + N.
  • · Keep symbols roughly the same size and aligned on a grid — examiners mark for clarity.
  • · Plan on paper first; rub out before committing — every line should land at one shape.
  • · Use connectors (A, B…) only if your diagram is large and arrows would cross the page.

How to read a flowchart

Trace method
  • · Treat it like a trace table — start at START and follow the arrows.
  • · When you reach a decision diamond, evaluate the condition with current values and follow the matching label.
  • · Each process box updates a variable — write the new value next to its column in your trace.
  • · Each I/O parallelogram: take the next input value from the question, or record the output in your OUTPUT column.
  • · Loops repeat the arrows back upwards — keep going around until the decision condition fails.
  • · Stop only when you reach STOP. Then list the final outputs in order.
Tip: mark each arrow you've travelled with a tick — it stops you re-entering a loop body in the wrong place.

Exam-style questions

Try first, then check

Cover the answer, attempt on paper, then expand.

Q1. Draw a flowchart that inputs a number and outputs 'positive', 'negative' or 'zero'. 6 marks

Mark scheme highlights — 1 mark each for: START/STOP ovals · INPUT parallelogram · first decision (N > 0) · second decision (N < 0) · three correct OUTPUTs · arrows joining all three branches to STOP.

flowchart TD A([START]) --> B[/INPUT N/] B --> C{N > 0?} C -- Yes --> D[/OUTPUT 'positive'/] C -- No --> E{N < 0?} E -- Yes --> F[/OUTPUT 'negative'/] E -- No --> G[/OUTPUT 'zero'/] D --> H([STOP]) F --> H G --> H
Q2. Describe what this flowchart does. 3 marks
flowchart TD A([START]) --> B[Largest ← 0] B --> C[I ← 1] C --> D{I <= 10?} D -- Yes --> E[/INPUT N/] E --> F{N > Largest?} F -- Yes --> G[Largest ← N] G --> H[I ← I + 1] F -- No --> H H --> D D -- No --> J[/OUTPUT Largest/] J --> K([STOP])

Model answer

  • · The flowchart inputs 10 numbers in turn. (1)
  • · It keeps track of the largest value seen so far in the variable Largest, updating it whenever a bigger number is read. (1)
  • · After all 10 numbers it outputs the largest number. (1)

Caveat: initialising Largest ← 0 would fail if all inputs are negative — a better version reads the first number into Largest before the loop. Mentioning this scores no extra marks but shows understanding.

Q3. Convert this flowchart to pseudocode. 5 marks

The flowchart inputs 30 marks, counts how many are a pass (≥ 50), and outputs the count.

flowchart TD A([START]) --> B[Pass ← 0] B --> C[I ← 1] C --> D{I <= 30?} D -- Yes --> E[/INPUT Mark/] E --> F{Mark >= 50?} F -- Yes --> G[Pass ← Pass + 1] G --> H[I ← I + 1] F -- No --> H H --> D D -- No --> J[/OUTPUT Pass/] J --> K([STOP])

Model pseudocode answer

DECLARE Mark, Pass, I : INTEGER
Pass ← 0                          // 1 mark — initialisation
FOR I ← 1 TO 30                  // 1 mark — count-controlled loop
  INPUT Mark                     // 1 mark — input inside loop
  IF Mark >= 50 THEN            // 1 mark — correct condition
    Pass ← Pass + 1
  ENDIF
NEXT I
OUTPUT Pass                       // 1 mark — output outside loop

Common slips: putting OUTPUT inside the loop, forgetting ENDIF/NEXT, or initialising Pass inside the loop (resets every iteration).

Last-minute

Six shapes only: oval (start/stop), rectangle (process), parallelogram (I/O), diamond (decision), circle (connector), arrow (flow). Wrong shape = lost mark. Every flowchart you draw in the exam should reach a STOP — no dead ends, no unlabelled arrows out of diamonds.

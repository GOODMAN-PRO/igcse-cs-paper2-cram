Searching Algorithms

Two methods you must know cold: Linear and Binary. Know the pseudocode, the trade-offs, and how to trace each by hand.

Linear Search

Sequential

Check each element from start to end, one at a time, until the target is found or the list ends.

  • Works on unsorted data — major advantage
  • Worst case: up to n comparisons (target last or absent)
  • Best case: 1 comparison (target first)
  • Time complexity (informal): linear — doubles list, doubles time
Pseudocode
Found ← False
Index ← 1
WHILE Index <= n AND Found = False DO
  IF List[Index] = Target THEN
    Found ← True
    Position ← Index
  ENDIF
  Index ← Index + 1
ENDWHILE
IF Found = True THEN
  OUTPUT "Found at position ", Position
ELSE
  OUTPUT "Not found"
ENDIF
Python equivalent
def linear_search(lst, target):
    for i, value in enumerate(lst):
        if value == target:
            return i  # index found
    return -1      # not found

Binary Search

Divide & Conquer

List MUST be sorted. Look at the middle item; eliminate half the list each step.

  • Precondition: data must be in order
  • Time complexity: O(log n) — much faster for large lists
  • 1024 items → at most 10 comparisons
  • 1,000,000 items → at most 20 comparisons
Pseudocode
Low ← 1
High ← n
Found ← False
WHILE Low <= High AND Found = False DO
  Mid ← (Low + High) DIV 2
  IF List[Mid] = Target THEN
    Found ← True
    Position ← Mid
  ELSE
    IF List[Mid] < Target THEN
      Low ← Mid + 1
    ELSE
      High ← Mid - 1
    ENDIF
  ENDIF
ENDWHILE
Python equivalent
def binary_search(lst, target):
    low, high = 0, len(lst) - 1
    while low <= high:
        mid = (low + high) // 2
        if lst[mid] == target:
            return mid
        elif lst[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1

Exam-Style Questions

Q1 — Write pseudocode for a linear search of an array of 15 integers for a target value. 5 marks

Mark scheme expects: input target, Found flag initialised False, loop 1 to 15, IF compare, set Found True & record position, output result for both cases.

DECLARE Numbers : ARRAY[1:15] OF INTEGER
DECLARE Target, Index, Position : INTEGER
DECLARE Found : BOOLEAN

// (Assume Numbers already populated)
OUTPUT "Enter target: "
INPUT Target

Found ← False
Index ← 1
WHILE Index <= 15 AND Found = False DO
  IF Numbers[Index] = Target THEN
    Found ← True
    Position ← Index
  ENDIF
  Index ← Index + 1
ENDWHILE

IF Found = True THEN
  OUTPUT "Found at position ", Position
ELSE
  OUTPUT "Not found"
ENDIF
Q2 — Explain why binary search cannot be used on an unsorted array. 2 marks

Model answer:

  • Binary search relies on comparing the target with the middle element and then discarding the half that cannot contain the target. (1)
  • If the array is not sorted, you cannot guarantee the target is in the lower or upper half from a single comparison, so the algorithm may eliminate the wrong half and miss the value. (1)
Q3 — State two advantages and one disadvantage of binary search compared to linear search. 3 marks

Advantages (any 2):

  • Faster on large lists — eliminates half each step (O(log n) vs O(n)).
  • Fewer comparisons on average and in the worst case.
  • More efficient when the same list is searched repeatedly.

Disadvantage (any 1):

  • Requires the data to be sorted first — sorting adds extra cost.
  • More complex to code (Low/High/Mid logic) and harder to debug.
  • For very small lists or unsorted data, linear is just as fast or faster overall.

Sorting — Bubble Sort

The only sort you must know for IGCSE. Get the nested loop, the swap with Temp, and the Swapped flag for early termination locked in.

How it works

  1. Go through the list comparing each adjacent pair.
  2. If a pair is out of order, swap them (using a Temp variable).
  3. After one full pass, the largest remaining unsorted value has "bubbled" to the end.
  4. Repeat: each subsequent pass needs one fewer comparison (the tail is already sorted).
  5. If a pass completes with no swaps, the list is sorted — exit early.

Why "bubble"? Large values rise to the top end of the list like air bubbles in water.

Pseudocode (with Swapped flag)
FOR i ← 1 TO n - 1
  Swapped ← False
  FOR j ← 1 TO n - i
    IF List[j] > List[j+1] THEN
      Temp ← List[j]
      List[j] ← List[j+1]
      List[j+1] ← Temp
      Swapped ← True
    ENDIF
  NEXT j
  IF Swapped = False THEN
    // Early termination — list is sorted
    EXIT FOR
  ENDIF
NEXT i

Note: n - i ensures we don't recompare items already in their final position.

Python equivalent
def bubble_sort(lst):
    n = len(lst)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if lst[j] > lst[j + 1]:
                # tuple swap — no Temp needed
                lst[j], lst[j + 1] = lst[j + 1], lst[j]
                swapped = True
        if not swapped:
            break  # early exit
    return lst

# Example
print(bubble_sort([5, 2, 4, 1]))
# → [1, 2, 4, 5]

Advantages
  • Very simple to code and explain.
  • In-place — no extra array needed (small memory use).
  • With Swapped flag, detects an already-sorted list in one pass.
  • Stable — equal items keep their relative order.
Disadvantages
  • Slow on large lists — worst-case O(n²).
  • Many comparisons even when only a few items are out of place.
  • Not used in production — quicksort/mergesort are faster.

Exam-Style Questions

Q1 — Write pseudocode for a bubble sort on an array of 10 integers in ascending order. 6 marks

Mark scheme: outer FOR loop, Swapped flag init, inner FOR loop with correct bounds (n - i), compare adjacent pair, swap using Temp, set Swapped True, early-exit on no swaps.

DECLARE Numbers : ARRAY[1:10] OF INTEGER
DECLARE i, j, Temp : INTEGER
DECLARE Swapped : BOOLEAN

// (Assume Numbers already populated)
FOR i ← 1 TO 9
  Swapped ← False
  FOR j ← 1 TO 10 - i
    IF Numbers[j] > Numbers[j+1] THEN
      Temp ← Numbers[j]
      Numbers[j] ← Numbers[j+1]
      Numbers[j+1] ← Temp
      Swapped ← True
    ENDIF
  NEXT j
  IF Swapped = False THEN
    EXIT FOR
  ENDIF
NEXT i

  • Advantage: Simple to understand and implement / requires no extra memory (in-place) / can detect an already-sorted list in one pass using the Swapped flag.

Standard Methods of Solution

Five tiny patterns you must be able to write from memory: Totalling Counting Max Min Average. Most Paper 2 questions combine 2–3 of these.

Totalling (Sum)

Σ

Add every input value to a running total. Always initialise Total to 0 before the loop.

Total ← 0
FOR i ← 1 TO 10
  INPUT Num
  Total ← Total + Num
NEXT i
OUTPUT Total

Counting (Conditional)

#

Increment a counter only when a condition is met. Initialise Count to 0.

Count ← 0
FOR i ← 1 TO 10
  INPUT Num
  IF Num > 50 THEN
    Count ← Count + 1
  ENDIF
NEXT i
OUTPUT Count

Finding Maximum

↑

Initialise Max to the first value, then compare every later value.

INPUT Max          // first value
FOR i ← 2 TO 10
  INPUT Num
  IF Num > Max THEN
    Max ← Num
  ENDIF
NEXT i
OUTPUT Max

Finding Minimum

↓

Same structure as Max — but compare with < Min.

INPUT Min          // first value
FOR i ← 2 TO 10
  INPUT Num
  IF Num < Min THEN
    Min ← Num
  ENDIF
NEXT i
OUTPUT Min

Finding the Average

Combine totalling with division by the count of inputs at the end. Be careful: divide after the loop, not inside it.

Total ← 0
FOR i ← 1 TO 10
  INPUT Num
  Total ← Total + Num
NEXT i
Average ← Total / 10
OUTPUT Average

Tip If the number of values is variable, count them in a separate counter and divide by that — never hard-code the divisor.

Combined: Max, Min & Average in One Pass

A typical Paper 2 question asks for multiple results from a single loop. Pattern: read first value → set Max and Min to it → loop the remaining values.

Total ← 0

// Read first value and use it to seed Max and Min
INPUT Num
Max ← Num
Min ← Num
Total ← Total + Num

FOR i ← 2 TO 10
  INPUT Num
  Total ← Total + Num
  IF Num > Max THEN
    Max ← Num
  ENDIF
  IF Num < Min THEN
    Min ← Num
  ENDIF
NEXT i

Average ← Total / 10

OUTPUT "Maximum: ", Max
OUTPUT "Minimum: ", Min
OUTPUT "Average: ", Average

Notice we only loop from 2 to 10 — the first value was already consumed before the loop. This is the safest pattern.

Important Gotcha

DANGER

Don't initialise Max to 0 (or Min to a huge number) — initialise to the FIRST value!

  • If you set Max ← 0 and all inputs are negative (e.g. temperatures -12, -5, -8), the algorithm reports 0 — which is not in the list and is wrong.
  • If you set Min ← 999 and all inputs are larger (e.g. 1200, 1500, 1100), the algorithm reports 999 — again not even a real input.
  • Fix: read the first input before the loop and assign it to both Max and Min. Then loop from index 2 onwards.
Wrong
Max ← 0
FOR i ← 1 TO 10
  INPUT Num
  IF Num > Max THEN
    Max ← Num
  ENDIF
NEXT i  // Breaks for all-negative input
Correct
INPUT Max
FOR i ← 2 TO 10
  INPUT Num
  IF Num > Max THEN
    Max ← Num
  ENDIF
NEXT i  // Robust

Exam-Style Questions

Q1 — Write pseudocode that inputs 20 numbers and outputs the largest and the average. Use a single loop. 6 marks

Mark scheme: initialise Total to 0, seed Max from first input, loop remaining 19, accumulate Total, conditional Max update, divide by 20 after loop, output both results.

DECLARE Numbers, Total, Max, Num, i : INTEGER
DECLARE Average : REAL

Total ← 0

// Seed Max with first value
INPUT Num
Max ← Num
Total ← Total + Num

FOR i ← 2 TO 20
  INPUT Num
  Total ← Total + Num
  IF Num > Max THEN
    Max ← Num
  ENDIF
NEXT i

Average ← Total / 20

OUTPUT "Largest: ", Max
OUTPUT "Average: ", Average

Watch out for: initialising Max to 0 (loses a mark on neg-only test data); dividing inside the loop; declaring Average as INTEGER (truncates).

Q2 — Write pseudocode that inputs marks for 30 students and outputs how many scored above 50. 4 marks

Mark scheme: Count initialised 0, loop 1 to 30, input Mark, IF condition strictly greater than 50, increment count, output count after loop.

DECLARE Count, Mark, i : INTEGER

Count ← 0

FOR i ← 1 TO 30
  OUTPUT "Enter mark for student ", i
  INPUT Mark
  IF Mark > 50 THEN
    Count ← Count + 1
  ENDIF
NEXT i

OUTPUT "Students scoring above 50: ", Count

Read the question wording carefully: "above 50" means > 50; "50 or above" means >= 50.

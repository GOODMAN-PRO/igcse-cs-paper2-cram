Pseudocode Reference (Cambridge style)

The exact Cambridge IGCSE 0478/0984 pseudocode conventions — keywords, syntax, and worked patterns you can lift straight into Paper 2 answers.

UPPERCASE keywords PascalCase identifiers Arrays start at 1 Use ← for assignment Indent loop & IF bodies

1. Variables & Constants

Declare every variable before use with DECLARE. Use ← for assignment, never =. Constants are fixed at declaration.

// Declaration: DECLARE name : TYPE
DECLARE Counter : INTEGER
DECLARE Price : REAL
DECLARE Initial : CHAR
DECLARE Name : STRING
DECLARE Found : BOOLEAN
DECLARE Birthday : DATE

// Assignment uses the left arrow
Counter ← 0
Price ← 4.99
Initial ← 'A'
Name ← "Alex"
Found ← FALSE
Birthday ← 19/05/2026

// Constants — value fixed at declaration
CONSTANT Pi = 3.14159
CONSTANT MaxSize = 100
CONSTANT Greeting = "Hello"

Tip Identifiers are case-sensitive in markschemes — stay consistent.

2. Input / Output

INPUT reads from the user; OUTPUT writes to the screen. Strings are joined with commas in OUTPUT lists.

// Plain input
INPUT Name
INPUT Age

// Input with a prompt (Cambridge accepts both forms)
OUTPUT "Enter your name: "
INPUT Name

// Simple output
OUTPUT Name
OUTPUT "The answer is ", Answer

// Multi-item output (comma-separated list)
OUTPUT "Hello, ", Name, "!"
OUTPUT "Score: ", Score, " out of ", Total

// Worked snippet
OUTPUT "Enter age: "
INPUT Age
OUTPUT "You are ", Age, " years old."

Note OUTPUT automatically moves to a new line at the end.

3. Selection (IF & CASE)

Two structures: IF…THEN…ELSE…ENDIF for binary/nested choices, CASE OF for many discrete values.

// Simple IF
IF Age >= 18 THEN
   OUTPUT "Adult"
ENDIF

// IF…ELSE
IF Mark >= 50 THEN
   OUTPUT "Pass"
ELSE
   OUTPUT "Fail"
ENDIF

// Nested IF — grade boundaries
IF Mark >= 80 THEN
   Grade ← 'A'
ELSE
   IF Mark >= 70 THEN
      Grade ← 'B'
   ELSE
      IF Mark >= 60 THEN
         Grade ← 'C'
      ELSE
         IF Mark >= 50 THEN
            Grade ← 'D'
         ELSE
            Grade ← 'F'
         ENDIF
      ENDIF
   ENDIF
ENDIF

// CASE — cleaner alternative
CASE OF Grade
   'A' : OUTPUT "Excellent"
   'B' : OUTPUT "Very good"
   'C' : OUTPUT "Good"
   'D' : OUTPUT "Pass"
   'F' : OUTPUT "Fail"
   OTHERWISE : OUTPUT "Invalid grade"
ENDCASE

// CASE with ranges and lists
CASE OF Mark
   80 TO 100 : Grade ← 'A'
   70 TO 79  : Grade ← 'B'
   60 TO 69  : Grade ← 'C'
   50 TO 59  : Grade ← 'D'
   OTHERWISE: Grade ← 'F'
ENDCASE

4. Iteration — three loop types

Cambridge expects you to choose the correct loop. FOR for known counts, WHILE for pre-test, REPEAT for post-test.

// FOR — count-controlled (known number of iterations)
FOR i ← 1 TO 10
   OUTPUT i
NEXT i

// FOR with STEP — count downwards or in fixed jumps
FOR i ← 10 TO 1 STEP -1
   OUTPUT i
NEXT i

FOR i ← 2 TO 20 STEP 2
   OUTPUT i        // 2, 4, 6, … 20
NEXT i

// WHILE — pre-test (may run zero times)
Total ← 0
INPUT Number
WHILE Number <> -1 DO
   Total ← Total + Number
   INPUT Number
ENDWHILE
OUTPUT "Total: ", Total

// REPEAT — post-test (always runs at least once)
REPEAT
   OUTPUT "Enter password: "
   INPUT Password
UNTIL Password = "secret123"

4b. When to use which loop

Pick by reading the question wording. Examiners look for the correct construct, not just a working answer.

  • FOR "do this 10 times", "for each pupil in class of 30", "every element of the array" — number of iterations is known.
  • WHILE "keep going while temperature is above 20", "until the user enters STOP" (test first). Condition may already be false ⇒ body skipped.
  • REPEAT "ask until input is valid", validation loops, menu prompts. Body must execute at least once.
  • Switch FOR → WHILE if the count depends on user input that you don't yet have.
// Validation pattern — REPEAT is ideal
REPEAT
   OUTPUT "Enter age (0-120): "
   INPUT Age
UNTIL Age >= 0 AND Age <= 120

// Sentinel pattern — WHILE is ideal
INPUT Value
WHILE Value <> 0 DO
   Sum ← Sum + Value
   INPUT Value
ENDWHILE

5. Arrays (1D & 2D)

Arrays are 1-indexed by default in Cambridge pseudocode. ARRAY[1:n] means indices 1 through n inclusive.

// 1-D array — 30 INTEGER marks
DECLARE Marks : ARRAY[1:30] OF INTEGER

// Assigning into elements
Marks[1] ← 75
Marks[2] ← 82
Marks[30] ← 60

// Reading every element with FOR
FOR i ← 1 TO 30
   OUTPUT "Enter mark ", i, ": "
   INPUT Marks[i]
NEXT i

// Summing array contents
Total ← 0
FOR i ← 1 TO 30
   Total ← Total + Marks[i]
NEXT i
Average ← Total / 30

// 2-D array — 5x5 grid of CHAR
DECLARE Grid : ARRAY[1:5, 1:5] OF CHAR

Grid[1, 1] ← 'X'
Grid[3, 4] ← 'O'

// Nested loop to fill a 2-D array
FOR Row ← 1 TO 5
   FOR Col ← 1 TO 5
      Grid[Row, Col] ← '.'
   NEXT Col
NEXT Row

// Searching a 1-D array (linear search)
Found ← FALSE
FOR i ← 1 TO 30
   IF Marks[i] = Target THEN
      Found ← TRUE
      Position ← i
   ENDIF
NEXT i

Watch out Using Marks[0] is out-of-bounds for ARRAY[1:30] — start at 1.

6. Procedures (no return value)

A procedure performs an action. Parameters are typed. Call with CALL and the procedure name.

// Procedure with one parameter
PROCEDURE PrintHello(Name : STRING)
   OUTPUT "Hello, ", Name
ENDPROCEDURE

// Calling it
CALL PrintHello("Alex")
CALL PrintHello("Sam")

// Procedure with no parameters
PROCEDURE DrawLine
   OUTPUT "----------"
ENDPROCEDURE

CALL DrawLine

// Procedure with multiple parameters
PROCEDURE PrintBill(Item : STRING, Price : REAL)
   OUTPUT Item, " costs $", Price
ENDPROCEDURE

CALL PrintBill("Coffee", 3.50)
CALL PrintBill("Cake", 4.25)

Key idea Procedures use CALL. Functions don't — they appear inside expressions.

7. Functions (returns a value)

A function computes and returns a value. Use it inside expressions or assignments — never with CALL.

// Function returning an INTEGER
FUNCTION Square(N : INTEGER) RETURNS INTEGER
   RETURN N * N
ENDFUNCTION

// Using it in an assignment
Answer ← Square(7)        // Answer = 49

// Using it inside an expression
Total ← Square(3) + Square(4)   // 9 + 16 = 25

// Using it inside an OUTPUT
OUTPUT "7 squared = ", Square(7)

// Function with REAL parameter and return
FUNCTION CircleArea(R : REAL) RETURNS REAL
   RETURN 3.14159 * R * R
ENDFUNCTION

Area ← CircleArea(5.0)

// Function returning BOOLEAN
FUNCTION IsEven(N : INTEGER) RETURNS BOOLEAN
   IF N MOD 2 = 0 THEN
      RETURN TRUE
   ELSE
      RETURN FALSE
   ENDIF
ENDFUNCTION

IF IsEven(Number) THEN
   OUTPUT "Even"
ENDIF

Common mistake Writing CALL Square(7) — wrong, it's a function so just use Square(7).

8. String Operations

Cambridge string functions are 1-indexed. Memorise the four below and the & concatenation operator.

// LENGTH — number of characters
DECLARE Word : STRING
Word ← "Computer"
N ← LENGTH(Word)          // N = 8

// LCASE — convert to lowercase
Lower ← LCASE("HELLO")     // "hello"

// UCASE — convert to uppercase
Upper ← UCASE("hello")     // "HELLO"

// SUBSTRING(s, start, length) — 1-indexed
Word ← "Computer"
Part ← SUBSTRING(Word, 1, 3)   // "Com"
Mid  ← SUBSTRING(Word, 4, 3)   // "put"
End  ← SUBSTRING(Word, 7, 2)   // "er"

// Concatenation with &
DECLARE FirstName : STRING
DECLARE LastName  : STRING
FirstName ← "Alex"
LastName  ← "Smith"
FullName ← FirstName & " " & LastName
// FullName = "Alex Smith"

// Worked example: extract first 3 chars
INPUT Word
IF LENGTH(Word) >= 3 THEN
   Code ← UCASE(SUBSTRING(Word, 1, 3))
   OUTPUT "Code: ", Code
ELSE
   OUTPUT "Word too short"
ENDIF

Indexing SUBSTRING("Hello", 1, 1) = "H" — the first character is at position 1, not 0.

9. Operators

Three families: arithmetic, comparison, logical. DIV and MOD always trip students up — see the table.

// Arithmetic: + - * / DIV MOD
A ← 7 + 2            // 9
B ← 7 - 2            // 5
C ← 7 * 2            // 14
D ← 7 / 2            // 3.5  (REAL division)

// DIV — integer division (drops remainder)
E ← 7 DIV 2          // 3
F ← 15 DIV 4         // 3
G ← 100 DIV 3        // 33

// MOD — remainder of integer division
H ← 7 MOD 2          // 1
I ← 15 MOD 4         // 3
J ← 100 MOD 3        // 1

// Comparison: returns BOOLEAN
//   =   equal
//   <>  not equal
//   <   less than
//   >   greater than
//   <=  less than or equal
//   >=  greater than or equal

IF Score >= 50 THEN ...
IF Name <> "Alex" THEN ...

// Logical: AND, OR, NOT
IF Age >= 13 AND Age <= 19 THEN
   OUTPUT "Teenager"
ENDIF

IF Day = "Sat" OR Day = "Sun" THEN
   OUTPUT "Weekend"
ENDIF

IF NOT Found THEN
   OUTPUT "Item missing"
ENDIF

Useful pattern N MOD 2 = 0 tests for even; N MOD 10 gives the last digit.

10. File Handling

Files must be opened in a mode, used, then closed. The three modes are READ, WRITE, APPEND.

// Open a file in one of three modes
OPENFILE "data.txt" FOR READ      // read existing
OPENFILE "data.txt" FOR WRITE     // overwrite/create
OPENFILE "data.txt" FOR APPEND    // add to end

// Read one line into a variable
READFILE "data.txt", LineVariable

// Write a line
WRITEFILE "data.txt", "text to write"

// Always close when done
CLOSEFILE "data.txt"

// EOF() returns TRUE when at end of file
// Full read-and-print pattern:
OPENFILE "names.txt" FOR READ
WHILE NOT EOF("names.txt") DO
   READFILE "names.txt", Line
   OUTPUT Line
ENDWHILE
CLOSEFILE "names.txt"

// Writing several lines
OPENFILE "log.txt" FOR WRITE
FOR i ← 1 TO 5
   WRITEFILE "log.txt", "Entry " & i
NEXT i
CLOSEFILE "log.txt"

// Append to an existing log
OPENFILE "log.txt" FOR APPEND
WRITEFILE "log.txt", "new event"
CLOSEFILE "log.txt"

Don't forget CLOSEFILE — easy mark lost in markschemes.

11. Data Types Deep Dive

Six built-in types. Watch the quotes: single for CHAR, double for STRING.

// INTEGER — whole numbers (positive, negative, zero)
DECLARE Score : INTEGER
Score ← 42
Score ← -7

// REAL — decimal/fractional numbers
DECLARE Temp : REAL
Temp ← 36.6
Temp ← -2.5
Temp ← 3.0      // still REAL, not INTEGER

// CHAR — exactly ONE character, single quotes
DECLARE Grade : CHAR
Grade ← 'A'
Initial ← 'Z'
Digit ← '7'     // the character '7', not the number

// STRING — text, double quotes
DECLARE Name : STRING
Name ← "Alex"
Message ← "Hello, world!"
Empty ← ""          // empty string is fine

// BOOLEAN — TRUE or FALSE only
DECLARE Active : BOOLEAN
Active ← TRUE
Active ← FALSE

// DATE — dd/mm/yyyy
DECLARE Today : DATE
Today ← 19/05/2026
Birthday ← 01/01/2010

Quote rule 'A' is CHAR. "A" is STRING. They're not the same type.

12. Operator Precedence

When a line mixes operators, evaluate in this order (highest first):

  1. ( ) — parentheses
  2. ^ — exponentiation (if used)
  3. *, /, DIV, MOD
  4. +, -
  5. = <> < > <= >= — comparison
  6. NOT
  7. AND
  8. OR
// Worked examples — left-to-right within a level
5 + 3 * 2             // = 5 + 6     = 11
(5 + 3) * 2           // = 8 * 2     = 16
20 - 6 / 2            // = 20 - 3    = 17
10 DIV 3 + 1          // = 3 + 1     = 4
10 MOD 3 * 2          // = 1 * 2     = 2

// Comparison + logical
A > 5 AND B < 10     // compare first, then AND
NOT X = Y               // equals first, then NOT

// Use brackets to make intent explicit
IF (Age >= 13) AND (Age <= 19) THEN
   OUTPUT "Teen"
ENDIF

Safe rule If in doubt, add brackets. Examiners reward clarity.

Bonus: Comments & Layout

Cambridge accepts both inline comments and well-formatted indentation. Add comments to earn structure marks.

// This is a single-line comment
Counter ← 0     // inline comment after a statement

// Indent the body of every block by ~3 spaces
IF A > B THEN
   OUTPUT A      // indented
ELSE
   OUTPUT B      // indented
ENDIF

// Nest indentation when blocks nest
FOR i ← 1 TO 5
   FOR j ← 1 TO 5
      IF i = j THEN
         OUTPUT i, ",", j
      ENDIF
   NEXT j
NEXT i

Bonus: Validation Patterns

Three checks Cambridge expects you to recognise. Use REPEAT for each.

// Range check — between two limits
REPEAT
   OUTPUT "Mark (0-100): "
   INPUT Mark
UNTIL Mark >= 0 AND Mark <= 100

// Length check — string of correct length
REPEAT
   OUTPUT "4-digit PIN: "
   INPUT Pin
UNTIL LENGTH(Pin) = 4

// Presence check — not empty
REPEAT
   OUTPUT "Name: "
   INPUT Name
UNTIL Name <> ""

// Type/format check via parsing isn't part of
// Cambridge pseudocode — assume INPUT gives
// you the declared type.

Quick-glance summary — all loop types side by side

Loop When test happens Min iterations Best for Syntax sketch
FOR Count is fixed 0 (if start > end) Known repetitions, array traversal FOR i ← 1 TO 10 … NEXT i
FOR STEP Count is fixed, custom step 0 Counting down, every Nth element FOR i ← 10 TO 1 STEP -1 … NEXT i
WHILE Before each iteration (pre-test) 0 Sentinel input, condition may already be false WHILE cond DO … ENDWHILE
REPEAT After each iteration (post-test) 1 (always) Input validation, retry menus REPEAT … UNTIL cond

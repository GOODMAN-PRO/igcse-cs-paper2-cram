Identify Errors in Code

A code-with-errors question is virtually guaranteed on Paper 2. You will be given a pseudocode or Python snippet containing 2–4 bugs and asked to identify the line, explain the bug, and write a corrected line. Work through these 10 problems — if you can fix all of them, you are exam-ready for this style of question.

Problem 1 — find the error(s)

Pseudocode that should sum 10 numbers and output the total.

// Sum 10 numbers
Total ← 0
FOR i ← 1 TO 10
  INPUT Num
  Total = Total + Num
NEXT i
OUTPUT Total
Show answer

Bug: Line 5 uses = for assignment. In pseudocode, = is only for comparison (equality). Assignment is always ←. The examiner will dock you a mark every time you do this.

Corrected line:

  Total ← Total + Num
Problem 2 — find the error(s)

Pseudocode that should find the largest of 5 input numbers (which may be negative).

Max ← 0
FOR i ← 1 TO 5
  INPUT Num
  IF Num > Max THEN
    Max ← Num
  ENDIF
NEXT i
OUTPUT Max
Show answer

Bug: Max is initialised to 0. If every input is negative (e.g. −5, −9, −2, −7, −1) then 0 stays as the maximum — but 0 was never actually input. The "max so far" must be a real input.

Fix: read the first number outside the loop, set Max to it, then loop 4 more times:

INPUT Num
Max ← Num
FOR i ← 2 TO 5
  INPUT Num
  IF Num > Max THEN Max ← Num
NEXT i
Problem 3 — find the error(s)

Python that should average 10 marks.

total = 0
for i in range(10):
    mark = input("Mark: ")
    total = total + mark
print(total / 10)
Show answer

Bug: input() always returns a string. Adding a string to the integer total raises TypeError: unsupported operand type(s) for +: 'int' and 'str'. You must cast to int.

Corrected line:

    mark = int(input("Mark: "))
Problem 4 — find the error(s)

A Python bubble sort attempt.

nums = [5, 2, 8, 1, 9]
for i in range(len(nums)):
    for j in range(len(nums)):
        if nums[j] > nums[j+1]:
            nums[j], nums[j+1] = nums[j+1], nums[j]
Show answer

Bug: When j = len(nums)-1 = 4, the line nums[j+1] accesses index 5 — out of range — raising IndexError. The inner loop must stop one short.

Fix:

    for j in range(len(nums) - 1 - i):

Subtracting i is also an efficiency improvement (the last i items are already sorted on each pass).

Problem 5 — find the error(s)

Pseudocode declaring and filling an array of 5 integers.

DECLARE Arr : ARRAY[1:5] OF INTEGER
FOR i ← 0 TO 5
  INPUT Arr[i]
NEXT i
Show answer

Two bugs:

  1. Pseudocode arrays declared as ARRAY[1:5] are 1-indexed. Arr[0] does not exist — accessing it is an error.
  2. The loop runs 0,1,2,3,4,5 — that is 6 iterations, but the array only has 5 slots.

Fix:

FOR i ← 1 TO 5
Problem 6 — find the error(s)

Pseudocode that should ask for a password and keep asking until the user gets it right.

REPEAT
  INPUT Password
UNTIL Password <> "secret"
Show answer

Bug: UNTIL in pseudocode means "loop ends when this condition becomes TRUE". The condition Password <> "secret" is true when the password is wrong, so the loop exits on the first wrong guess — the opposite of what was wanted.

Fix:

UNTIL Password = "secret"

Remember: REPEAT … UNTIL <exit condition>. The condition is what makes the loop stop, not continue.

Problem 7 — find the error(s)

Python that should sum the integers 1 to 10 inclusive (answer should be 55).

total = 0
for i in range(1, 10):
    total = total + i
print(total)  # prints 45, expected 55
Show answer

Bug: range(a, b) in Python is exclusive of b. range(1, 10) gives 1, 2, …, 9 — sum is 45. To include 10, the upper bound must be 11.

Fix:

for i in range(1, 11):

Contrast with pseudocode: FOR i ← 1 TO 10 is inclusive of both ends.

Problem 8 — find the error(s)

Pseudocode that should output the numbers 1 to 5.

FOR i ← 1 TO 5
  OUTPUT i
ENDIF
Show answer

Bug: The loop is closed with ENDIF. That keyword closes an IF … THEN block. A FOR loop closes with NEXT i.

Fix:

NEXT i

Quick reference of closers: IF → ENDIF, FOR → NEXT i, WHILE → ENDWHILE, REPEAT → UNTIL, CASE → ENDCASE, PROCEDURE → ENDPROCEDURE, FUNCTION → ENDFUNCTION.

Problem 9 — find the error(s)

An SQL query that should return all electronics products.

SELECT * FROM PRODUCT WHERE Category = Electronics;
Show answer

Bug: Electronics with no quotes is interpreted by SQL as a column name, not a string literal. Either the query fails (no such column) or it does something completely unintended.

Fix: wrap text values in quotes.

SELECT * FROM PRODUCT WHERE Category = "Electronics";

Numbers (Price > 50) and booleans (InStock = TRUE) are not quoted. Only text and dates.

Problem 10 — find the error(s)

Pseudocode that validates that an age is between 0 and 120 inclusive.

INPUT Age
IF Age < 0 OR Age > 120 THEN
  OUTPUT "Valid"
ELSE
  OUTPUT "Invalid"
ENDIF
Show answer

Bug: The condition Age < 0 OR Age > 120 is true when the age is out of range — i.e. invalid. The outputs are swapped.

Fix: swap the outputs, or invert the condition:

IF Age >= 0 AND Age <= 120 THEN
  OUTPUT "Valid"
ELSE
  OUTPUT "Invalid"
ENDIF

Note: an age range check is range validation, not length or type. State the right validation name in 1-mark questions.

Common Mistakes Examiners Penalise

These are the recurring deductions seen in every examiner report. Most of them lose only 1 mark each — but a paper with five of these slips is the difference between an A and an A*. Read all of them tonight.

Mistake 1

Using = for assignment in pseudocode

Pseudocode uses ← for assignment and = only for comparison. Writing x = 5 in pseudocode is wrong even if "everyone knows what you mean".

// Wrong
Count = Count + 1
// Right
Count ← Count + 1
Mistake 2

Forgetting to initialise Total or Count before the loop

Adding to a variable that was never set is undefined behaviour. Always set running totals and counters to 0 (or 1) before the loop begins.

// Wrong — Total used before being set
FOR i ← 1 TO 10
  Total ← Total + i
NEXT i

// Right
Total ← 0
FOR i ← 1 TO 10
  Total ← Total + i
NEXT i
Mistake 3

Off-by-one in FOR i ← 1 TO 9

Pseudocode TO is inclusive on both ends. FOR i ← 1 TO 10 is 10 iterations, FOR i ← 1 TO 9 is only 9. Python range(1, 10) is also only 9. Always check whether the question wanted "the first 10" or "1 to 10".

Mistake 4

Not casting int(input()) in Python

A bare input() returns a string. If you intend to do arithmetic or numeric comparison, you must cast.

# Wrong
n = input("Enter: ")
if n > 10: ...   # TypeError

# Right
n = int(input("Enter: "))
Mistake 5

Omitting validation when the question explicitly asks for it

If the question says "the program must validate the input is between 1 and 100", you must include an IF (or REPEAT … UNTIL) check — not just accept input. State the validation type in commentary where possible (range, length, type, presence, format, lookup/check digit).

Mistake 6

Forgetting quotes around SQL text values

Text and dates need quotes; numbers and booleans don't. Single or double quotes both work, but be consistent within one query.

-- Wrong
SELECT * FROM Student WHERE Name = Smith;
-- Right
SELECT * FROM Student WHERE Name = "Smith";
Mistake 7

Trace tables: one row per line instead of per change

A trace table records the value of each variable when it changes. You do not write a new row for OUTPUT or for an IF evaluation that doesn't update anything. Output usually has its own column.

Mistake 8

Confusing validation and verification

Validation = is the data sensible? (range, length, type, presence, format, lookup) — done by the computer. Verification = was the data copied/entered correctly? (double-entry, visual check) — about transcription, not sense.

Mistake 9

Bubble sort without a Swapped flag

The full mark scheme version of bubble sort uses a Swapped boolean so the algorithm can exit early. Forgetting it usually still scores most marks — but if the question says "most efficient" or "stop when sorted", you must include it.

REPEAT
  Swapped ← FALSE
  FOR i ← 1 TO n - 1
    IF A[i] > A[i+1] THEN
      // swap
      Swapped ← TRUE
    ENDIF
  NEXT i
UNTIL Swapped = FALSE
Mistake 10

Returning the value when asked for the position

For linear/binary search questions, read whether the output should be the item found, its index/position, or just a boolean "yes/no found". The three are different answers.

Mistake 11

Forgetting CLOSEFILE / f.close()

Any file question expects the file to be closed at the end. It's a free mark — never skip it.

OPENFILE "data.txt" FOR READ
  READFILE "data.txt", Line
CLOSEFILE "data.txt"
Mistake 12

Writing Python elif in pseudocode

Pseudocode does not have elif. Use nested IF … ELSE IF, or much better, CASE OF … ENDCASE for multi-way selection.

// Multi-way preferred form
CASE OF Grade
  "A" : OUTPUT "Excellent"
  "B" : OUTPUT "Good"
  "C" : OUTPUT "Pass"
  OTHERWISE : OUTPUT "Fail"
ENDCASE
Mistake 13

Initialising Max with 0 (fails for negative inputs)

The safe pattern is to initialise Max with the first input value, then loop over the rest. The reverse mistake is initialising Min with 0 when inputs may be positive.

Mistake 14

Misreading <= as < in range checks

"Between 1 and 10 inclusive" means >= 1 AND <= 10, not > 0 AND < 11 (which is the same, but harder to read), and definitely not > 1 AND < 10 (which excludes 1 and 10). Always re-check the boundary.

Mistake 15

Pseudocode arrays treated as 0-indexed

A pseudocode array declared ARRAY[1:n] starts at index 1, not 0. Loops should start at 1. Confusing Python's 0-indexed lists with pseudocode's 1-indexed arrays is one of the most common errors on Paper 2.

Mistake 16

SQL: ORDER BY placed before WHERE

The clause order is fixed: SELECT … FROM … WHERE … GROUP BY … ORDER BY …. Putting them in a different order is a syntax error.

-- Wrong order
SELECT Name FROM Student ORDER BY Name WHERE Year = 11;
-- Right
SELECT Name FROM Student WHERE Year = 11 ORDER BY Name;
Mistake 17

Forgetting the colon in Python control statements

Every Python if, for, while, def, and else must end with a colon. Indentation alone is not enough.

# Wrong
if x > 10
    print(x)

# Right
if x > 10:
    print(x)
Mistake 18

Mixing single and double quotes inconsistently

Python accepts both 'x' and "x" for strings, but mixing them in the same string literal (e.g. "hello') is a syntax error. In pseudocode, use double quotes consistently — that is what the mark scheme uses.

Mistake 19

Confusing MOD / DIV with /

DIV is integer division (drop the remainder). MOD is the remainder. / is true (float) division. Python equivalents: // and %. Mixing these up turns "is it even?" into nonsense.

7 DIV 2 = 3      // integer quotient
7 MOD 2 = 1      // remainder
7 / 2     = 3.5    // real division
Mistake 20

Not ending with an OUTPUT

If a question says "calculate X" or "find Y", the marker almost always expects you to display the result. A program that calculates the right answer but never outputs it loses 1 mark every time.

Exam Strategy

  • First 3 minutes: read the whole paper. Flag any question that looks hard with a tiny star — you'll do those last so you don't burn time staring at them.
  • Use pencil for flowcharts and trace tables. Mistakes are inevitable; ink corrections look messy and lose method-mark legibility.
  • Long answers (programming questions worth 6+ marks): begin with a one-line comment stating the plan, e.g. // initialise total, loop 10 times reading input, output total. The examiner gives method marks for clear intent even if the code has slips.
  • Trace tables: draw a thin horizontal line every time the loop restarts. It prevents you from skipping rows or duplicating values when there are several iterations to record.
  • Always end programming answers with OUTPUT (or print()). Calculating the right value but never displaying it is a near-universal mark deduction.
  • Running out of time? Write a pseudocode skeleton: INPUT → IF or LOOP → the calculation → OUTPUT. Even a 4-line skeleton typically scores 2–3 method marks — far better than zero.
  • Never leave a question blank. If you have any idea, write it. The mark scheme rewards "shows attempt to use a loop", "identifies need for validation", etc., even on partly-wrong answers.
  • For SQL: check clause order before you write the next clause. WHERE before GROUP BY before ORDER BY. Single statement, ends with a semicolon.
  • For Python answers: remember the colon, remember to indent, remember to cast inputs. Three free marks lost there in a typical paper.
  • State the validation type by name (range / length / type / presence / format / check digit). Markers look for the word.
  • After every section, glance back at the question. Did it ask for position or value? Count or list? Many slips come from answering a slightly different question.

Mental Math Refresher

DIV & MOD

Integer arithmetic for "every nth", "last digit", "is even", etc.

7 DIV 2 = 3     10 DIV 3 = 3     25 DIV 10 = 2
7 MOD 2 = 1     10 MOD 3 = 1     25 MOD 10 = 5
// Python equivalents: //  and  %

Even number test: n MOD 2 = 0. Last digit: n MOD 10. Drop last digit: n DIV 10.

Binary basics (mostly Paper 1, occasionally useful here)
8-bit place values:  128 64 32 16  8  4  2  1
10101010           = 128+32+8+2 = 170
11111111           = 255
10000000           = 128

Hex: 0–9 then A=10, B=11, … F=15. FF = 255. One hex digit = 4 bits.

Common ASCII values
'A' = 65     'a' = 97     '0' = 48     space = 32

Letter to lowercase: add 32. Digit char to number: subtract 48 (or the ASCII code of '0'). Useful if a Caesar-cipher or character-shift question comes up.

Quick conversions to memorise
1 KB =  1024 bytes        1 byte = 8 bits
1 MB =  1024 KB
1 GB =  1024 MB
